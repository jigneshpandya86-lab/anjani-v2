## Core Architecture & Stack
1. **Frontend**: Code is hosted on GitHub. Verify and test the frontend build before committing.
2. **Backend**: Powered by Firebase. The active Firebase project is **`anjaniappnew`**. All cloud functions, hosting, and Firestore rules must deploy here.
3. **SMS Mechanism**: SMS notifications are triggered and sent using the **Macrodroid Webhook URL** (e.g. configured in Firestore or env variables).

## Backend Cloud Functions Directory
All backend Cloud Functions are located in the [functions/](file:///home/jigneshpandya86/anjani-v2/functions) folder.
* **Main Script File**: The functions are defined and exported in [functions/index.js](file:///home/jigneshpandya86/anjani-v2/functions/index.js). To edit or add cloud functions, modify this file.
* **Dependencies**: Any backend npm package dependencies (such as `@google-cloud/vertexai` or `firebase-admin`) must be declared in [functions/package.json](file:///home/jigneshpandya86/anjani-v2/functions/package.json).

### Existing Exported Cloud Functions:
* **Triggered Functions**:
  * `sendSmsViaMacrodroid`: Document trigger on `leads/{docId}`. Sends SMS when a lead is created.
  * `sendOrderSmsToStaff`: Document trigger on `orders/{docId}`. Sends order notifications to staff on writes.
* **Scheduled Functions (`onSchedule` hourly/daily)**:
  * `sendMorningOrderReminders`: Daily morning scheduled reminders for today's orders.
  * `sendEveningOrderReminders`: Daily evening scheduled reminders for tomorrow's orders.
  * `sendWeeklyPaymentReminders`: Hourly checker function. Reads `config/defaulterReminder` to send weekly payment reminders to defaulters.
  * `sendWeeklyRegularOrderReminder`: Hourly checker function. Reads `config/regularReminder` to send weekly order reminders to regular clients.
  * `hourlyDeliveryReminders`: Hourly delivery checks and reminders.
  * `scheduleDueFollowUps`: Handles processing due lead follow-ups.
  * `sendStaffLeadReminders`: Reminds staff of leads to follow up on.
  * `sendIntelligentOrderReminders`: Intelligent order timing notifications.
  * `intelligentStaffTaskReminders`: [Disabled/Removed] Daily scheduled reminders for staff tasks.
  * `discoverLeadsWithAI`: AI-based crawler to search and discover new leads.
* **Callable Functions (`onCall` HTTPS endpoints)**:
  * `getNotifications`: Fetches system notifications.
  * `sendNotification`: Sends FCM/system notifications.
  * `registerDevice`: Saves device FCM tokens for notifications.

## Backend & Database Practices
* **Firestore Configuration Documents**:
  * `config/defaulterReminder`: Contains schedule config for weekly payment reminders.
  * `config/regularReminder`: Contains schedule config for regular client reminders.
* **Gemini AI Prompts**: Use `@google-cloud/vertexai` (Gemini model e.g. `gemini-1.5-flash`) inside backend scheduled functions to generate polite weekly SMS templates in a warm mix of Hindi and Gujarati.
* **Function Scheduling**: Schedule functions to run hourly (e.g. `0 * * * *`) and inspect the active configuration documents to determine if SMS dispatch is enabled for the current hour and weekday.
